R: Lessons Learned

General Thoughts:

Pro: Subsetting dataframes in R is easier than it is in Pandas.

Con: Documentation of R tools is less beginner-friendly.

Con: R Studio files do not convert directly into html pages for use in my portfolio, like Jupyter Notebooks do. Jupyter Notebooks can use an R kernel, but setting it up took days, and cells had to be run one at a time. Running the entire notebook with an R kernel hung up my system.

Every tool I used in my R project had an equivalent tool in Pandas.

Conclusion: For developing a portfolio that I can publish online, Jupyter/Pandas is more convenient. For exploring data and creating graphs, R and R Studio is more convenient.

pd.df.loc

Very fast when selecting rows based on labels.

Better at handling complex queries

df.loc[row_label]
df.loc[[row_label1, row_label2]]
df.loc[df['shield'] > 6]
df.loc[df['shield'] > 6, ['max_speed']] #returns just the 'max_speed' column
df.loc[(df['max_speed'] > 1) & (df['shield'] < 8)]
df.loc[(df['max_speed'] > 4) | (df['shield'] < 5)]

Brackets

Useful for selecting just rows based on complex criteria.

above_35 = titanic[titanic["Age"] > 35]
class_23 = titanic[titanic["Pclass"].isin([2, 3])]
class_23 = titanic[(titanic["Pclass"] == 2) | (titanic["Pclass"] == 3)]

Masking

I think this is just another name for using brackets.

Cleaning Dataframes

df.info()

Lists dataframe columns with their data types and non-null values

df.duplicated().sum()

Counts duplicate rows. Can subset by column or index.

df.col.nunique()

Counts the number of unique values in column.

df.col.unique()

Lists unique values in column.

df.sample(5)

Takes a random sample of 5 rows.
